# Lux-Auto SSO Implementation Guide

## Project Overview

**Project**: Lux-Auto - Automated arbitrage opportunity finder with SSO authentication  
**Architecture**: OAuth2 SSO (Google) + FastAPI Backend + PostgreSQL + Docker Compose  
**Deployment**: kubernetes/Docker Compose on GCP with Caddy reverse proxy  
**Status**: SSO Epic Implementation (Issues #63-#71)

## Code Style & Standards

### Python (FastAPI Backend)
- **Formatter**: Black (configured in settings.json)
- **Linter**: Pylint + flake8
- **Type Hints**: Required for all functions
- **Docstrings**: Google-style docstrings for all public functions
- **Testing**: Pytest with 90%+ coverage target
- **Async**: Use async/await for I/O-bound operations

### TypeScript/JavaScript (Frontend)
- **Formatter**: Prettier
- **Linter**: ESLint with standard config
- **Type Checking**: No strict mode enforced yet

### Shell Scripts
- **Style**: POSIX-compliant (bash 3.2+)
- **Error Handling**: `set -euo pipefail` for safety
- **Logging**: Colored output with log/warning/error functions
- **Idempotency**: Scripts should be safe to run multiple times

### Infrastructure (YAML/Terraform)
- **Format**: 2-space indentation
- **Comments**: Extensive documentation for non-obvious config
- **Validation**: `terraform fmt` and `docker-compose config` before commit

## Architecture Pattern

### OAuth2 Flow
```
User → Caddy (HTTPS) → oauth2-proxy (auth) → FastAPI (app) → PostgreSQL (data)
```

### Service Stack
1. **Caddy** (Port 80/443) - Reverse proxy, HTTPS termination, security headers
2. **oauth2-proxy** (Port 4180) - Google OAuth2 auth middleware
3. **FastAPI** (Port 8000) - Application backend, REST API
4. **PostgreSQL** (Port 5432) - User data, sessions, application data

### Database Models

```python
# Auth-related models
class UserRole(str, Enum):
    ADMIN = "admin"      # Full access
    USER = "user"        # Standard access
    VIEWER = "viewer"    # Read-only

class User(Base):
    id: int
    email: str (unique, indexed)
    role: UserRole
    active: bool
    created_at: datetime
    last_login: datetime

class UserSession(Base):
    user_id: int (FK to users)
    token_hash: str (SHA256, indexed)
    issued_at: datetime
    expires_at: datetime
    ip_address: str
    user_agent: str
    active: bool
    revoked_at: datetime (nullable)
```

## Secret Management

### Google Secret Manager (GSM) Secrets
All production secrets stored in GSM project `lux-auto`:

```
1. lux-google-oauth-client-id       → GOOGLE_CLIENT_ID
2. lux-google-oauth-client-secret   → GOOGLE_CLIENT_SECRET
3. lux-oauth2-cookie-secret         → OAUTH2_PROXY_COOKIE_SECRET (base64)
4. lux-fastapi-secret-key           → FASTAPI_SECRET_KEY (base64)
5. lux-db-password                  → DATABASE_PASSWORD
6. lux-db-connection-string         → DATABASE_URL
```

### Local Development
- **Never commit `.env`** (use `.env.example` for template)
- **Never commit credentials** (only placeholders in examples)
- Use `scripts/fetch-gsm-secrets.sh` to populate `.env`

## Security Considerations

### Authentication
- **OAuth2**: Google OAuth2 only (no passwords stored)
- **Session TTL**: 24 hours (configurable)
- **Token Storage**: SHA-256 hashed in database (never plaintext)
- **Admin Identification**: Hardcoded email in UserService.ADMIN_USER_EMAIL

### Network
- **HTTPS**: Enforced via Caddy (HTTP → HTTPS redirect)
- **HSTS**: max-age=31536000 (1 year)
- **CSP**: Restrictive Content-Security-Policy headers
- **CORS**: Origin restricted to lux.kushnir.cloud

### Data Protection
- **Secrets**: Never in code, only in GSM or environment
- **Logs**: No credentials/tokens logged
- **Audit**: Security events tagged with [SECURITY] prefix
- **Cleanup**: Expired sessions deleted every 6 hours

## Troubleshooting

### Service Health Issues
```bash
# Quick health check
docker compose ps

# Detailed service logs
docker compose logs -f oauth2-proxy | head -50

# Direct service testing
curl https://lux.kushnir.cloud/health
curl https://lux.kushnir.cloud/ping
docker exec lux-postgres pg_isready -U lux_admin
```

### OAuth Not Working
1. Verify GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET in .env
2. Check DNS resolves: `nslookup lux.kushnir.cloud`
3. Verify Caddy cert: `docker exec lux-caddy caddy list-certs`
4. Check oauth2-proxy logs: `docker compose logs oauth2-proxy`

### Database Connection Issues
```bash
# Check PostgreSQL logs
docker compose logs postgres

# Connect to DB directly
docker exec -it lux-postgres psql -U lux_admin -d lux_prod

# Run migrations
docker compose exec -T fastapi alembic upgrade head
```

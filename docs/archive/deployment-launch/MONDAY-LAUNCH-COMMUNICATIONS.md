## SECTION 5: Contingency Communications

### If Critical Blocker Emerges (Use as-needed)

#### Template A: Blocker Alert (If team can't start)
**Channel:** Slack #blockers  
**Audience:** Tech lead, Program lead, DevOps  
**Sender:** Team member with blocker  
**Urgency:** IMMEDIATE  

```
🚨 BLOCKER: [What is blocked]

**Issue:** [Description of problem]
**Impact:** [What can't proceed without this]
**Attempted fixes:** [What you already tried]
**Request:** [Specific help needed]

Example:
🚨 BLOCKER: Team can't clone repository - GitHub access denied for 3 members

**Issue:** Sarah, Alex, Jordan get "access denied" when cloning lux-auto
**Impact:** Can't start Monday - no local code to work with
**Attempted fixes:** Checked their GitHub accounts exist, tried re-adding to repo
**Request:** Can you verify their write access and regenerate tokens if needed?
```

---

#### Template B: Blocker Status Update (Every 30 mins if ongoing)
**Channel:** Slack #blockers  
**Audience:** Relevant parties  
**Sender:** Tech lead handling blocker  

```
🔄 BLOCKER UPDATE: [Issue name]

**Status:** Still investigating / Narrowed down / Ready to implement fix

**Latest finding:** [What we learned]

**Next step:** [What happens in next 30 min]

**ETA to resolution:** [Estimate when fixed]

Example:
🔄 BLOCKER UPDATE: GitHub access denied for 3 team members

**Status:** Found the issue! 
**Latest finding:** Their GitHub accounts hadn't been added to lux-auto team yet. Admin oversight.
**Next step:** Adding them to org now (via GitHub UI) - should take 10 min
**ETA to resolution:** 10:30 AM PT (in 20 minutes)

#### Template C: Blocker Resolved (Celebratory!)
**Channel:** Slack #blockers  
**Audience:** Team  
**Sender:** Tech lead who fixed it  

```
✅ BLOCKER RESOLVED: [Issue name]

**What was wrong:** [Root cause]
**How we fixed it:** [Solution applied]
**What didn't work:** [Dead ends, for learning]
**Time to resolution:** [From alert to fixed]

**Team impact:** Can now proceed with [work that was blocked]

Example:
✅ BLOCKER RESOLVED: GitHub access denied for 3 team members

**What was wrong:** Team members weren't added to lux-auto organization
**How we fixed it:** Added 3 members to org via GitHub settings -> Members
**What didn't work:** They could see the repo URL but org settings blocked clone access
**Time to resolution:** 37 minutes (8:53 AM alert, 9:30 AM fixed)

Learning: Always verify GitHub org access before team starts. Added to checklist.

## Notes for Communicators

**Tone Guidance:**
- Confident (we've planned extensively)
- Supportive (team is learning, support is real)
- Energetic (this is exciting!)
- Clear (no ambiguity, specific times/actions)
- Accountable (leadership is present, problems get solved)

**Avoid:**
- Vague timelines ("soon," "eventually")
- Negative tone ("be careful," "probably fail")
- Unclear instructions (always have specific actions/links)
- Over-communication (respect people's time)

**Safe to assume:**
- Team has pre-work assignments
- Team has GitHub access (verified Friday)
- Team has read documentation
- Team wants to succeed (they're here!)
- Leadership is supportive (verified in gut-checks)

**Customize before sending:**
- [Names] - Replace with actual person names
- [Links] - Replace with actual Slack links / Zoom links / repo paths
- [Time] - Adjust for your timezone (examples use PT)
- [Metrics] - Update with actual data Friday/Monday
- Tone - Match your team's communication style

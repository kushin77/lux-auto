## How GitHub Issues Works (You'll Use This)

### Creating an Issue
1. Go to GitHub → Issues → New Issue
2. Select appropriate template (Feature)
3. Fill in title and description
4. Add label `phase-4`
5. **Create Issue**

→ **System auto-assigns it to you** ✅

### Creating a PR for Your Issue
1. Create branch locally: `git checkout -b feature/health-check-api`
2. Make your code changes
3. Commit locally
4. Push to origin: `git push origin feature/health-check-api`
5. Create PR with **one of these in the description:**
   - "Fixes #61"
   - "Closes #62"
   - "Resolves #63"
6. Request review
7. **Click to merge**

→ **System auto-closes the issue** ✅  
→ **System auto-updates the epic** ✅  

### What You DON'T Have to Do
- ❌ Manually assign issues
- ❌ Manually manage status labels
- ❌ Manually close issues
- ❌ Manually update epic progress
- ❌ Manually write weekly status

The system does all of that automatically.
